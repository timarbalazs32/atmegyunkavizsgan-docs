Backend Dokumentáció

Ezen az oldalon a Node.js alapú backend fejlesztéshez kapcsolódó adatbázis-kezelési megoldásokat találod meg, különös tekintettel az SQLite3 variánsokra.

1. SQLite3 minta (Aszinkron / Callback alapú)

Az alapvető sqlite3 driver használata Express.js környezetben. Ez a könyvtár aszinkron módon, callback függvények segítségével kezeli a kéréseket, és modern import (ESM) szintaxist használ.

index.js (sqlite3) 
import express from 'express';
import cors from 'cors';
import sqlite3 from 'sqlite3';

const app = express();
const port = 8000;

app.use(cors());
app.use(express.json());

const db = new sqlite3.Database(`sakila.db`);

app.get('/', (req, res) => {
    res.json({ message: 'Welcome to the Film Portal Backend' });
});

app.get(`/films`, (req, res) => {
    try {
        let { limit, offset } = req.query;
        if (!limit || !offset) { limit = 50; offset = 0; }
        if (typeof limit !== "number" || typeof offset !== "number") {
            return res.status(400).json({ error: "The parameters have invalid datatypes" });
        }
        const sql = `select * from film`;
        db.all(sql, [], function(error, rows) {
            if (error) { return res.status(400).json({ error: error.message }); }
            return res.status(200).json(rows);
        });
    } catch (error) {
        return res.status(400).json({ error: error.message });
    }
});

app.listen(port, () => {
    console.log(`Server is running on port ${port}`);
});

2. Better-SQLite3 minta (Szinkron / Mozi hálózat kezelő API)

Ez a példa a better-sqlite3 csomagot használja szinkron módon egy mozinézési és filmnyilvántartó rendszerhez. VIZSGATIPP: Figyelj a metódusokra! Több soros válaszhoz .all(), egyetlen sorhoz .get(), adatmódosításhoz (INSERT, DELETE, UPDATE) pedig .run() kell.

index.js (better-sqlite3) 
const cors = require('cors');
const express = require('express');
const Database = require('better-sqlite3');

const app = express();
app.use(express.json());

app.use(cors());

const db = new Database('mozihalo.db');

app.get("/", (req, res) => {
  res.json({ message: "Üdvözlés" });
});

app.get("/filmek", (req, res) => {
  try {
    const sql = "select * from filmek";
    const result = db.prepare(sql).all(); 
    return res.status(200).json(result);
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

app.get("/vetites/:vazon", (req, res) => {
  try {
    const vazon = req.params.vazon;
    const sql = `
      SELECT filmek.fcim, mozik.mnev, mozik.varos 
      FROM vetitesek 
      INNER JOIN mozik ON mozik.mazon = vetitesek.mazon 
      INNER JOIN filmek ON filmek.fazon = vetitesek.fazon 
      WHERE vetitesek.vazon = ?`;
    
    const result = db.prepare(sql).get(Number(vazon)); 
    return res.status(200).json(result);
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

app.post("/filmek", (req, res) => {
  try {
    const { fazon, fcim, rendezo, gyartasi_ev, mufaj, jegypont } = req.body;
    const sql = "insert into filmek (fazon, fcim, rendezo, gyartasi_ev, mufaj, jegypont) values (?, ?, ?, ?, ?, ?)";
    
    const result = db.prepare(sql).run(fazon, fcim, rendezo, gyartasi_ev, mufaj, jegypont);
    return res.status(201).json({ message: "Film sikeresen hozzáadva", id: result.lastInsertRowid });
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

app.delete("/film/:fazon", (req, res) => {
  try {
    const fazon = Number(req.params.fazon);
    const sql = "delete from filmek where fazon = ?";
    db.prepare(sql).run(fazon); 
    return res.status(200).json({ message: "Film sikeresen törölve" });
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

app.post("/mozik", (req, res) => {
  const { mazon, mnev, varos, termek_szama } = req.body;
  try {
    const sql = "insert into mozik (mazon, mnev, varos, termek_szama) values (?, ?, ?, ?)";
    const result = db.prepare(sql).run(mazon, mnev, varos, termek_szama);
    return res.status(201).json({ message: "Mozi sikeresen hozzáadva", id: result.lastInsertRowid });
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

app.patch("/vetites/:vazon", (req, res) => {
  try {
    const vazon = Number(req.params.vazon);
    const { fazon, mazon, kezdete } = req.body;
    const sql = "update vetitesek set fazon = ?, mazon = ?, kezdete = ? where vazon = ?";
    db.prepare(sql).run(fazon, mazon, kezdete, vazon); 
    return res.status(200).json({ message: "Vetítés sikeresen frissítve" });
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

const PORT = 8000;
app.listen(PORT, () => {
  console.log(`Szerver fut: http://localhost:${PORT}`);
});

3. Tanuló Szerviz megvalósítása (tanulo.service.ts)

tanulo-service.ts 
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

export interface Tanulo {
  oktazon: number;
  nev: string;
  hozott: number;
  kpmagy: number;
  kpmat: number;
}

@Injectable({
  providedIn: 'root',
})
export class TanuloService {
  private apiUrl = 'http://localhost:8000/tanulok';
  constructor(private http: HttpClient) {}

  getOsszesTanulo(): Observable {
    return this.http.get(this.apiUrl);
  }

  deleteTanulo(oktazon: number): Observable {
    return this.http.delete(`http://localhost:8000/tanulo/id/${oktazon}`);
  }

  addTanulo(tanulo: Tanulo): Observable {
    return this.http.post("http://localhost:8000/tanulo", tanulo);
  }
}

Reszponzív formázás (Tailwind CSS)

A modern webfejlesztés elengedhetetlen része az eszközökhöz alkalmazkodó elrendezés. A Tailwind a Mobile-First elvet követi.

Tailwind Play CDN (Beillesztés a Head részbe)

Ezt a script sort kell bemásolnod az index.html fájlod <head> részébe, hogy azonnal működjenek a Tailwind osztályok külön telepítés nélkül:

HTML Head részlet 
<script src="https://cdn.tailwindcss.com"></script>

1. Alapvető Breakpointok (Képernyőméretek)

A Tailwind-ben a reszponzivitást előtagokkal kezeljük. Az alap osztályok mobilon érvényesek, a nagyobb méretekre a töréspontokkal ugrunk rá:

  • sm: $\ge 640\text{px}$ – Nagyobb mobilok / Kisméretű tabletek
  • md: $\ge 768\text{px}$ – Tabletek (A vizsgák leggyakoribb töréspontja)
  • lg: $\ge 1024\text{px}$ – Laptopok, normál asztali monitorok
  • xl: $\ge 1280\text{px}$ – Nagy felbontású monitorok

2. Gyakori Reszponzív Minták & Best Practice

A) Flexbox irányváltás (Pl. Menüsorok): Mobilon az elemek egymás alatt vannak (függőlegesen), míg asztali gépen egymás mellett sorakoznak.

class="flex flex-col md:flex-row justify-between"

B) Dinamikus Grid oszlopszámok (Pl. Termékkártyák): Mobilon egyetlen oszlopban, tableten kettőben, nagy kijelzőn pedig négy oszlopban rendeződnek az elemek.

class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4"

C) Intelligens szélességek (Widths): Kerüld a fix százalékokat mobilon! Mobilon legyen teljes szélességű a container, és csak nagyobb kijelzőn feleződjön meg.

class="w-full md:w-1/2 mx-auto"
