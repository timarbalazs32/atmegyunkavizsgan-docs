Backend Dokumentáció

Ezen az oldalon a Node.js alapú backend fejlesztéshez kapcsolódó adatbázis-kezelési megoldásokat találod meg, különös tekintettel az SQLite3 variánsokra.

1. SQLite3 minta (Aszinkron / Callback alapú)

Az alapvető sqlite3 driver használata Express.js környezetben. Ez a könyvtár aszinkron módon, callback függvények segítségével kezeli a kéréseket, és modern import (ESM) szintaxist használ.

index.js (sqlite3) 
import express from 'express';
import cors from 'cors';
import sqlite3 from 'sqlite3';

const app = express();
const port = 8000;

app.use(cors());
app.use(express.json());

const db = new sqlite3.Database(`sakila.db`);

app.get('/', (req, res) => {
    res.json({ message: 'Welcome to the Film Portal Backend' });
});

app.get(`/films`, (req, res) => {
    try {
        let { limit, offset } = req.query;
        if (!limit || !offset) { limit = 50; offset = 0; }
        if (typeof limit !== "number" || typeof offset !== "number") {
            return res.status(400).json({ error: "The parameters have invalid datatypes" });
        }
        const sql = `select * from film`;
        db.all(sql, [], function(error, rows) {
            if (error) { return res.status(400).json({ error: error.message }); }
            return res.status(200).json(rows);
        });
    } catch (error) {
        return res.status(400).json({ error: error.message });
    }
});

app.listen(port, () => {
    console.log(`Server is running on port ${port}`);
});

2. Better-SQLite3 minta (Szinkron / Mozi hálózat kezelő API)

Ez a példa a better-sqlite3 csomagot használja szinkron módon egy mozinézési és filmnyilvántartó rendszerhez. A projekt hagyományos CommonJS (require) modulfelkéréssel épül fel.

index.js (better-sqlite3) 
const cors = require('cors');
const express = require('express');
const Database = require('better-sqlite3');

const app = express();
app.use(express.json());

app.use(cors());
const db = new Database('mozihalo.db');

app.get("/", (req, res) => {
  res.json({ message: "Üdvözlés" });
});

app.get("/filmek", (req, res) => {
  try {
    const sql = "select * from filmek";
    const result = db.prepare(sql).all();
    return res.status(200).json(result);
  } catch (error) {
    return res.status(500).json(error.message);
  }
});

app.listen(8000, () => {
  console.log(`Szerver fut: http://localhost:8000`);
});

3. Tanuló Szerviz megvalósítása (tanulo.service.ts)

A szerviz felel az aszinkron API hívások (GET, POST, DELETE) lebonyolításáért és az adatok típusosságáért (Interface):

tanulo-service.ts 
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

// Tanuló adatstruktúra leírása
export interface Tanulo {
  oktazon: number;
  nev: string;
  hozott: number;
  kpmagy: number;
  kpmat: number;
}

@Injectable({
  providedIn: 'root',
})
export class TanuloService {

  private apiUrl = 'http://localhost:8000/tanulok';

  constructor(private http: HttpClient) {}

  // Összes tanuló lekérése (GET)
  getOsszesTanulo(): Observable {
    return this.http.get(this.apiUrl);
  }

  // Tanuló törlése azonosító alapján (DELETE)
  deleteTanulo(oktazon: number): Observable {
    return this.http.delete(`http://localhost:8000/tanulo/id/${oktazon}`);
  }

  // Új tanuló rögzítése (POST)
  addTanulo(tanulo: Tanulo): Observable {
    return this.http.post("http://localhost:8000/tanulo", tanulo);
  }
}

4. Fő lista komponens (tanulok.ts & tanulok.html)

Ez a komponens kéri le az adatokat a szerviztől az ngOnInit életciklus-módszerben, majd a HTML-ben végigiterál rajtuk:

tanulok.ts 
import { Component, OnInit } from '@angular/core';
import { TanuloService, Tanulo } from '../../services/tanulo-service';
import { CommonModule } from '@angular/common';
import { TanuloCard } from "../tanulo-card/tanulo-card";

@Component({
  selector: 'app-tanulok',
  standalone: true,
  imports: [CommonModule, TanuloCard],
  templateUrl: './tanulok.html',
  styleUrl: './tanulok.css',
})
export class Tanulok implements OnInit {
  TanulokLista: Tanulo[] = [];

  constructor(private tanuloService: TanuloService) {}

  ngOnInit() {
    this.tanuloService.getOsszesTanulo().subscribe((data) => {
      this.TanulokLista = data;
      console.log(this.TanulokLista);
    });
  }
}

6. Adatfelvételi Űrlap (Template-driven Form)

Új adatok rögzítése a backend felé az Angular FormsModule csomagjának segítségével. VIZSGATIPP: A HTML-ben az input mezőknek kötelező megadni a name attribútumot és a ngModel direktívát, különben az Angular nem fogja beolvasni az értéküket!

tanulo-form.ts 
import { Component } from '@angular/core';
import { FormsModule } from '@angular/forms'; // <-- KÖTELEZŐ import a formokhoz!
import { TanuloService } from '../../services/tanulo-service';

@Component({
  selector: 'app-tanulo-form',
  standalone: true, // Modern Angular 17+ komponens jelleg
  imports: [FormsModule], // <-- Be kell tenni az imports tömbbe is!
  templateUrl: './tanulo-form.html',
  styleUrl: './tanulo-form.css',
})
export class TanuloForm {

  // A szerviz bejekttálása a konstruktorba, hogy elérjük az addTanulo metódust
  constructor(private TanuloService: TanuloService) {}

  // Az űrlap elküldésekor lefutó függvény (a HTML-ből kapja meg a form referenciát)
  onSubmit(form: any) {
    // form.value tartalmazza az input mezők értékeit kulcs-érték párokként (pl: { oktazon: ..., nev: ... })
    console.log("Űrlap aktuális értéke:", form.value);

    // Meghívjuk a szerviz POST kérését és feliratkozunk rá (subscribe)
    this.TanuloService.addTanulo(form.value).subscribe({
      next: (response) => {
        console.log("Sikeres hozzáadás a szerverhez:", response);
        form.reset(); // VIZSGATIPP: Sikeres mentés után ürítsük ki az űrlapot!
      },
      error: (err) => {
        console.error("Hiba történt a mentés során:", err);
      }
    });
  }
}
